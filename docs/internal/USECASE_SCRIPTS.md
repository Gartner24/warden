# Use-Case Scripted Walkthroughs

## UC-01: Beacon Flood Detection (offline)

**Actor:** Operator / analyst  
**Precondition:** Python venv active, `tests/fixtures/pcap/beacon-flood.pcap` present

1. Start detector against the beacon-flood fixture:
   ```
   python3 -m detector.main --pcap tests/fixtures/pcap/beacon-flood.pcap \
       --bssid AA:BB:CC:DD:EE:FF --ssid LAB_WARDEN_UTP
   ```
2. Observe stdout: `BEACON_FLOOD` alert emitted with `severidad: ALTA`.
3. Confirm session summary shows `alertas_totales >= 1`.

**Expected result:** At least one `BEACON_FLOOD` alert, no false `DEAUTH` or `EVIL_TWIN`.

## UC-05: Defender Panel - WebSocket Alert Stream

**Actor:** Operator via browser  
**Precondition:** Detector server running (`uvicorn detector.web.server:app`)

1. Open `http://localhost:8000` in a browser.
2. Connect WebSocket client (built into the panel UI).
3. POST `/api/detector/start` with target BSSID and SSID via the UI.
4. Feed a PCAP that generates alerts (or use live capture on an interface in monitor mode).
5. Observe real-time alert cards appear in the Defender Panel.

**Expected result:** Alerts pushed via WebSocket without page reload; each card shows tipo, severidad, timestamp, and detalles.

## UC-12: Full Chain Attack - E2E Offline Demo

**Actor:** Evaluator / CI  
**Precondition:** `chain.pcap` fixture present (generated by `scripts/generate-fixtures.py`)

1. Run the demo script:
   ```
   scripts/demo-uc12.sh
   ```
2. Script starts the detector against `chain.pcap`.
3. Detector processes all three attack phases in order: Beacon Flood -> Deauth -> Evil Twin.
4. Correlator fires `CADENA_OFENSIVA` once all three phases are observed within the correlation window.
5. Output printed to stdout; script greps for all 4 alert types.

**Expected result:**
```
[demo] Alert types detected:
BEACON_FLOOD
CADENA_OFENSIVA
DEAUTH
EVIL_TWIN
[demo] Done.
```

All 4 alert types present. Exit code 0.

**Automated equivalent:** `tests/test_acceptance_chain.py::test_acceptance_chain_triggers_all_alert_types`
